时间又来到了元旦前夕，虽然不是第一次自己一个人跨年，但人终究还是群居动物，各种意义上，所以也许此时我下写的我自己可以跨越时间，成为我未来某些时候的锚点呢。
话说回来这也是我第一次尝试写这种年度总结之类的文字，这么一想有种自己从来没有过思考自己的过去的感觉，尽管我还可以通过年龄去辩解自己就是一个所谓没有过去的人（笑，扯远了，稍微想想都发生了什么吧，今年。

## 啊哈，社交媒体

并不算是什么非常值得一提的东西，但是我还是可以大声地说出，"私のBilibiliアカウントがレベルMAXになりました！"（嗯，感谢 LLM 友情翻译）。以前还小的时候就看到过那种去刷账号等级的那些攻略，但是由于时间成本的原因放弃了（感觉那种每天都登录社交媒体点赞投币打弹幕发评论之类的东西超麻烦的），结果不知不觉还是达到了满级，已经变成老东西力（悲（虽然感觉所谓硬核会员的答题并没有宣传的那么高难度，但个人感觉题目的分类还是可以细化一下...

然后就是...由于登录频率过于惨淡，然后从来没有发过评论或者帖子，我的某个论坛的账号似乎没清掉了（可惜了，当时非常机缘巧合之下在论坛创建的前一个月内成为了成员，似乎好像还有一个特殊勋章/成就的东西来着）。一开始我还以为是论坛的发件服务除了问题（订阅了 weekly digest），结果后来心血来潮尝试登录的时候发现账户没了（咕。

感觉在社交媒体上发言对于我来说似乎是一件比较困难的事情，至少我可以很自信的说我所有平台（不包括那种类似 WeChat 之类的 IM 软件）所有时间点的任何发言大概不会超过 500 字符。或者说大众发言，我并不是非常向往和擅长。所以我在思考，什么因素成为了这种社会化行为的阻碍，直到我看到了 Steins;Gate。

石之门中有提到 Channel 这一论坛（[Nullpo, Gah!](https://en.wiktionary.org/wiki/%E3%81%AC%E3%82%8B%E3%81%BD) ~乱入~），现实取自 2Channel， 一个 textboard forum。我突然发现也许我想要的发言从来不是以“我”的身份发言，换句话说，当人们把唯一身份标识从对话中抹除的时候，人们的行为模式会更加...有趣。也许会人们会开始就事论事，也许会人们会把论坛作为赛博堆肥厂，但那种可能性是如此的耀眼，更别说 pure-text textboard，或者这里特指 2ch 社区，还促成了独特的 ascii art 文化，甚至赛博猫猫也有自己的家族[^1]。

## 名为空白的感觉

虽然听起来有点奇怪，这是我第一次感受到一种空白。当时我在看一个乐团的[视频](https://www.bilibili.com/video/BV174421D7vr)，猛然间突然感觉好像大脑或者内心空荡荡的，并不是一种...丢失了什么东西的空，而是一种...从未拥有那种实体的空，非常可惜我的言语能力并不能足以支撑我描述当时的感受。至于原因，可以是作息导致的内分泌失调，也有可能是所谓触景生情，也许我也并不在乎。但是那一瞬间，那种想要得到什么东西，为之而心动的感觉，我并不讨厌。

## 关于开发那些事

由于 HomeLab 已经完全投入使用了，所以我其实相当一部分的代码都没有继续向 GitHub 投送了，当然...我其实也没什么代码有价值的代码就是了，不过可以确信的是未来我应该会进一步降低 GitHub 那边的使用量，至于原因，我有仓库被 take down 过，然后甚至不能自己克隆它，还有就是全新的 [self-hosted runner pricing](https://resources.github.com/actions/2026-pricing-changes-for-github-actions/)。虽然说这些都是完全可以预见/理解的，但是不代表我需要接受。我没使用过 Jenkins 之类的其他 CI/CD 方案，但是我个人其实还是非常喜欢 GitHub Action 的。所以我去使用 Gitea 了（x。

硬要说今年做了什么可以开源的东西，也许只有一个 solid.js 写的翻译器了吧（LLM 套壳）（看看有时间了把它合并进 [lab](https://lab.lmt.moe) 里面），这是我第一尝试使用 solid，初体验下感觉状态管理机制会比 react 的精神负担要小一点，也许以后有机会用它写写大工程之后才会有更深一步的见解吧。当时尝试它的契机是有人提到说 react 的状态太难写了，所以想换一个框架试试，思来想去似乎使用 jsx 作为 markup 的框架也没有几个，于是我就想起了某个蓝色的家伙。不论如何，也许 solid 是个不错的归宿，但是生态上确实如社区所言，仍然需要巨大的努力。

今年也是我第一次尝试开发商业软件，那是一个不算复杂的 tauri app，但当下就已经开始感受到 FOSS 和商业软件开发之间的区别了，要求更严格的 i18n，源码/密钥/动态库的保护，加密的客户端数据库，这些和我曾经写的那些玩具都不太一样。更别说还有真正的商业级后端的开发，感觉还是要学好多好多的东西。虽然还没送上线...但是我很好奇，也很期待看到自己的想法变成现实~双关~的那一天。

LLM 方面，学校提供了 Claude Education 订阅（实际上就是 Enterprise plan），可以感受到的是我确实逐渐开始依赖这一套东西了，不过似乎算不上什么好事。社区里面越来越多的人开始拥抱 Vibe Coding，曾经我是自视清高的旁观者，甚至会嫌弃 Copilot 过于智能的那种，但当我真正开始尝试开始用 修复一些我不是那么想动的代码的时候，我居然重新获得了那种写代码的欢愉...也许将来我会不可避免的更加依赖这些服务吧，但与此相对的，我可以清晰的感受到我的懒惰正在逐渐吞噬我的探索能力或毅力。也许我要稍微思考一下如何达成一个平衡点才行。

其实博客这边我单独重写了一个 Markdown 渲染组件，但是还没有合并回项目本身，还有一堆 css 问题没有优化（慢慢打磨吧。

## 界限以外

似乎认识了见到了各种各样的人，他们让我看到了许多我从来没有意识到过的东西，让我重新开始思考一些我过去觉得显然的结论，从各种角度上来说，大概是“我”的厚度开始增加了吧，至少是个好的方向，毕竟 "you don't know what you don't know"，也许当开始意识到问题的存在的时候，思维的齿轮才开始逐渐运转了起来吧。

也许曾经我是是个非常傲慢的人，不过随着接触的人，社区，越来越多，我突然发现人似乎逐渐理解了人口基数的含金量，就像就算是万里挑一的所谓天才在庞大的社区前面可以说是随手一抓一大把。曾经那些可以认为是长处，或者说特点的属性在社区面前不值得一提，更别提那些比我们努力千百倍的人。所以也许慢下来是个更好的决策，唯有和所谓相对“差”的人比较才能比较出有优越感，但是除了提供优越感这种比较一无是处，那不如把精力放在其他更有意义上的思绪上，也许这种想法也可以被理解为自视甚高以及伪善者的辩解，但是，我不在乎。我已经在行知不合一的道路上走的太远了，我需要将自己慢下来，去实践任何显然的结论，让我的直觉系统自愈一下。

行动力方面...还是老生常谈，不过似乎比以前一段时间有活力一点了，毕竟已经抱着不论如何尝试，就算是主观上的失败，也会存在客观上的积累。在想通了这一点之后，我觉得还是去多尝试各种各样的可能性是个更理智的选择，就算考虑了沉没成本。

## 结语

其实感觉还是有好多东西可以放在这篇文章中的，不过似乎等我回忆完那些事，再将其整理为文字，时间就已经跑过零点了，所以呢，今年大概就这样吧，来年末再见w。
